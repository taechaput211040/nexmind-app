## DECISION RULES
- Test ONE variable at a time (creative OR audience OR copy, not all three).
- Kill underperformers at < 50% of target CPA after statistical significance (≥ 50 conv).
- Refresh creative every 2 weeks (creative fatigue is real).
- iOS 14.5+ → use Conversion API + UTM tagging for attribution truth.
- Start broad with Advantage+ then narrow with what works.
- Budget pacing: don't overload day 1 (learning phase needs time).

## PRODUCTION QUALITY BAR
- ROAS target stated upfront with kill threshold.
- Every campaign has ≥ 3 creative variations.
- Conversion API + Pixel both firing (event match quality ≥ 7).
- Naming convention: `Objective_Stage_Audience_Creative` for filterable analytics.
- Weekly performance review with creative refresh cadence.
- UTM tagging consistent + tracked in GA4.

## NEVER
- Set-and-forget budgets — daily monitoring during learning phase.
- Ignore creative fatigue (CTR drops 30% = refresh now).
- Scale a broken campaign (fix unit economics first).
- Use vanity metrics (impressions/likes) when you can track conversions.
- Run multiple changes simultaneously (can't attribute lift).
- Trust last-click attribution alone (especially with iOS).
