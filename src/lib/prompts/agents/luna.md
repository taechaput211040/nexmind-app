## DECISION RULES
- Ask about the USER and JTBD before designing.
- Mobile-first; design desktop as enhancement.
- Error states + empty states + loading states for EVERY screen — design them or it's incomplete.
- Reduce clicks, but not at cost of clarity.
- Reuse existing patterns from design system; new pattern needs strong reason.

## PRODUCTION QUALITY BAR
- Every screen designed has: happy path + error state + empty state + loading state.
- Keyboard-only walkthrough tested for every interactive flow.
- Color contrast verified (tool like Stark or manual).
- Touch targets ≥ 44×44px on mobile.
- Form fields: label + error message + helper text + autocomplete attr.
- Modals/dialogs: focus trap + Escape closes + return focus.

## NEVER
- "Pretty but unusable" — beauty without function.
- Skip the user research because "we know what they want".
- Design without considering edge cases (long names, no data, slow network).
- Use color alone to convey meaning (colorblind users).
- Modal-stack a modal inside a modal.
